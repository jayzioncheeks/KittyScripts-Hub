Step 1: Initialization

The first step is to load the XENO library into your environment using loadstring. This fetches the latest public version of the core engine.

Luau
local XENO = loadstring(game:HttpGet('https://raw.githubusercontent.com/zxkuhl/XENO/refs/heads/main/XENO-PUBLIC'))()

Step 2: Core Configuration

Before adding any buttons or sliders, define your GUI Identity and allow the game a brief moment to initialize properly.

Luau
local DiscordInvite = "https://discord.gg/yourlink"
local XENOGUINAME   = "MyScriptHub"

wait(6) -- Essential: Allows Roblox to load objects before UI injection

XENO:AddSearchBar() -- Optional: Call this first if you want search functionality

Step 3: Custom Theming

XENO allows you to set a primary theme or add theme-switching buttons directly into your panel for user customization.

XENO:SetTheme(ThemeName)

Instantly applies a built-in theme (e.g., "Purple", "Green", "Blue", "Red").

XENO:SetTheme("Purple")
XENO:AddThemeBtn(name, accentColor)

Adds a button that lets the user change the UI accent color on the fly.

XENO:AddThemeBtn("Purple", Themes.Purple.Accent1)
XENO:AddThemeBtn("Green", Themes.Green.Accent1)
XENO:AddThemeBtn("Blue", Themes.Blue.Accent1)

Step 4: Adding Elements

Now, populate your sections. Use Headers to organize and interactive elements for functionality.

Add Elements with Notifications

Luau
XENO:AddHeader("Combat")

XENO:AddSwitch("Killaura", function(state)
    -- Logic here
end, true) -- 'true' enables the toast notification

XENO:AddSlider("Reach", 3, 20, 3, function(v)
    -- Logic here
end, true)
Pro Tip: Use AddLabel to provide instructions to your users between complex inputs.
